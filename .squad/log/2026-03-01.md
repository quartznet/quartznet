## What Happened

Hockney and Verbal jointly debugged and fixed `Quartz.Tests.AspNetCore` test failures caused by the `.sln`→`.slnx` solution format migration. The team discovered that `WebApplicationFactory` requires explicit content-root configuration when the default `*.sln` lookup fails.

**Test Status Before:** All 51 AspNetCore tests failing (InvalidOperationException: Solution root could not be located)  
**Test Status After:** All 51 AspNetCore tests passing  

**Secondary Issue Found:** VBCSCompiler lock contention during unit test runs (CS2012 error). Verbal confirmed the lock is environmental, not code-related. Mitigation: disable shared compilation and build in isolation.

## Key Fixes Applied

1. **AspNetCore test content root:** Set `ASPNETCORE_TEST_CONTENTROOT_QUARTZ_TESTS_ASPNETCORE` environment variable before `WebApplicationFactory` instantiation.
2. **Build isolation:** Use `/p:UseSharedCompilation=false -m:1` if VBCSCompiler lock persists.
